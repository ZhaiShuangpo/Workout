作为一个健身爱好者，在选择或使用一款健身Web应用时，最核心的诉求通常是**“低摩擦的数据录入”**、**“科学的反馈循环”**以及**“高度的自定义灵活性”**。

以下是为您梳理的Web应用功能清单及具体需求描述：

---

### 一、 核心功能模块：训练计划管理 (Training Module)


| 功能名称 | 需求描述 |
| :--- | :--- |
| **动态训练计划创建** | 用户可按“周”或“天”维度制定计划。支持添加动作、设置目标组数、重量和次数。需支持**“动作库关联”**，点击动作名可直接查看技术要点。 |
| **训练实时记录 (Workout Log)** | 训练进行时，支持一键填报。具备**“自动倒计时器”**功能（每组结束点击完成，自动触发组间休息闹钟）。支持**“RPE/RIR记录”**，即感知负荷强度录入。 |
| **动作替换/随机调整** | 考虑到健身房器械排队情况，允许用户在不破坏计划的前提下，快速替换相似动作（如：杠铃深蹲替换为哑铃高位深蹲）。 |
| **渐进性超负荷提醒** | 系统自动对比上周同动作数据。在本周训练开始前，提示用户“建议增加2.5kg”或“建议多做1次”，辅助用户实现长期进步。 |

---

### 二、 核心功能模块：饮食与营养追踪 (Nutrition Module)



| 功能名称 | 需求描述 |
| :--- | :--- |
| **能量代谢计算器 (TDEE)** | 基于用户的身高、体重、体脂率及活动等级，自动计算基础代谢及每日目标摄入热量（热量盈余或缺口）。 |
| **三大营养素配比 (Macros)** | 允许用户自定义碳水、蛋白质、脂肪的比例。进度条实时显示：**已摄入 vs 剩余可摄入**。 |
| **补剂补给提醒** | 针对肌酸、蛋白粉、多维等健身人群常备补剂，设置每日摄入打卡功能，确保摄入的一致性。 |
| **自定义菜谱库** | 允许用户录入常用的健身餐配置（如“鸡胸肉+西蓝花+糙米饭”），下次录入时可一键添加，无需重复计算。 |

---

### 三、 辅助功能模块：数据分析与反馈 (Analytics)

| 功能名称 | 需求描述 |
| :--- | :--- |
| **力量增长曲线** | 自动提取核心动作（深蹲、硬拉、卧推等）的 **1RM (估计最大重量)** 变化趋势，并生成可视化折线图。 |
| **身体成分追踪** | 记录体重、体脂率、围度（胸围、臂围、腰围等）。支持上传**对比照片**，按日期横向排列。 |
| **训练负荷统计** | 按周统计训练总量（Volume = 重量 × 组数 × 次数），帮助用户判断是否过度训练或训练不足。 |

---

### 四、 用户体验 (UX) 关键需求描述

* **极简录入模式：** 健身时手部有汗或手握器械，页面应尽量采用大按钮，减少打字录入，多用增减控件（+/-）。
* **模版快速复制：** 绝大多数人每周的训练是循环的。需要支持“一键复制上周计划”到本周的功能。
* **离线缓存支持：** 部分健身房信号较差，应用应具备基本的本地暂存功能，网络恢复后自动同步数据。
* **深色模式支持：** 考虑到许多人有晨练或晚练习惯，深色模式对眼部更友好。
